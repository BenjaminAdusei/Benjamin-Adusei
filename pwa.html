📋 BSTR PWA

Basic School Terminal Report – Offline‑first Progressive Web App for Ghanaian teachers

🚀 Launch Live Demo

The Problem

Every term, Ghanaian basic school teachers manually compute grades, write remarks, calculate school fees, and print terminal reports for 30–50 students. This process is:

  • Time‑consuming – many hours of repetitive arithmetic.
  • Error‑prone – miscalculated totals or inconsistent remarks.
  • Internet‑dependent – many teachers lack reliable connectivity for cloud tools.

As a teacher myself, I knew there had to be a better way.

My Solution: BSTR PWA

A fully offline, installable Progressive Web App that runs entirely in the browser. Teachers can open it once, install it to their home screen, and use it without internet forever.

Key differentiator: No internet required after installation. All data stays on the teacher’s device (private and free). One‑time deployment, zero recurring server costs.

🌐 PWA & Distribution Features

  • Installable PWA – Works offline after first visit; can be added to home screen on Android, iOS, Windows, Mac.
  • Custom App Icon – Uses 192×192 and 512×512 icons.
  • Service Worker Caching – Caches all assets; fully functional without internet.
  • Netlify One‑Click Deploy – Drag‑and‑drop deployment; automatic HTTPS and global CDN.
  • Password‑Protected QR Code – Optional secure access; share only with authorised teachers.

🎨 Design & Branding

  • Ghana‑flag inspired header (green, yellow, red accents).
  • Print‑friendly: headers & footers printed in white background with black text for ink saving.
  • Trademark “bksolutions” prominently displayed in footer (screen and print).
  • Clear section titles: “ACADEMIC PERFORMANCE”, “CONDUCT & REMARKS”, “BILLS FOR NEXT TERM”.

🔧 Developer & Business Features

  • No external backend – All data stored locally. Zero server costs, works offline forever.
  • Easy white‑labelling – Can be rebranded for individual schools (custom header, logo, colours).
  • One‑time payment model – Suitable for selling per teacher or per school license.
  • No app store approval – Instant distribution via URL or QR code; installs directly from browser.

Technologies Used

HTML5/CSS3 JavaScript (Vanilla) localStorage API Service Workers PWA Manifest Netlify Hosting

Impact & Results

During a pilot with 8 teachers in two basic schools:

  • Average time to generate a single terminal report dropped from ~12 minutes to under 2 minutes.
  • Teachers reported zero arithmetic errors in grades and fee calculations.
  • The offline nature was praised – one teacher used it successfully during a 3‑day power and internet outage.
  • Custom subject addition allowed schools to adapt the tool to their unique curricula instantly.

What I Learned

Building for offline‑first environments requires designing with constraints in mind. localStorage is surprisingly robust for single‑user educational tools. Also, teachers value choice over automation – dropdown remarks work better than fully auto‑generated text, because human nuance still matters.
